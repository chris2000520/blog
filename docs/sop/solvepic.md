# 个人博客图片存储问题
之前搭建博客的时候，只顾美化主题，一些普通的逻辑结构没有完成，就去整那些花里胡哨的主页，实在有点本末倒置，美化的一个大组成成分就是图片，而图片一般比代码和文章要大多了，就导致每次加载网页时都时非常慢的。而我们写思路的的时候，又不得不去使用一些图片，我所知道的有两种方式。

## 本地上传
当我们只需要使用少量图片时，可以本地上传，在文章中的markdown语法中，我们可以使用
```
![缩略图](/images/pic.png)
```
这样的语法，来插入图片。[]里的文字就是当图片加载出问题时，显示的文字，后面的链接是主题配置文件里的source/images/文件名.png。
## 使用图床
知乎上有个大佬写的各种各样的图床的一篇文章，[盘点国内免费好用的图床](https://zhuanlan.zhihu.com/p/35270383)。可以借鉴一下，然后呢，我一个一个试了好多，有的需要钱，有的只有几个月，要不就是很复杂，我呢，只想简简单单在描述算法或解决问题的方法时偶尔添加一些图片来加强理解，思来想去我就使用了这个，[路过图床](https://imgchr.com)。

## 更新
由于担心图床跑路，图片加载也不是很快，我看别人的博客加载很快（[SlovinG的博客](http://sloving.gitee.io/)），完全看不到加载项，原理是直接部署到Gitee码云代码仓库里，国内的Gitee码云速度是真的快，然而我迁移后，发现不能绑定个人域名，详细请看这篇[博客](https://yyqx.online/posts/%E7%BD%91%E7%AB%99%E8%BF%81%E7%A7%BB%E8%87%B3gitee/)。

然而我还是舍不得我用了好久的域名，然后我就用Coding了，图片都上传到Coding上，毕竟是国内的网站，加载图片还是可以的。

小测试：

由于评论区是Valine，语法支持Markdown，所以只需要输入一下代码就可以获得一只猫,快来试试！
```Markdown
[![Nssgde.th.png](https://s1.ax1x.com/2020/06/26/Nssgde.th.png)](https://imgchr.com/i/Nssgde)
```
